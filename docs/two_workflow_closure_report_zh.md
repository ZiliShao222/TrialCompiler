# TrialCompiler 双工作流阶段性验收报告

## 1. 验收结论

TrialCompiler 当前已经形成两条可执行工作流：

1. **既有临床试验文件的一致性审核工作流**：以 NCT04683926 / OMNI-PAIN-103 为公开测试案例，对 Protocol、SAP、ICF 和注册资料执行事实提取、冲突识别、变更传播与治理检查。
2. **受控临床试验方案生成工作流**：以 Metformin-PAD 历史截断测试包为案例，按 Phase 1 证据归纳、Phase 2 增量修订、Evaluator-only 盲评三个可见性阶段运行。

两条工作流在工程上均已跑通，但专业效果尚未达到可直接用于临床、统计、注册或质量决策的程度。当前定位应为：**可复现的研究原型与 benchmark harness，而不是已验证的临床文档产品。**

## 2. NCT04683926 一致性审核工作流

### 2.1 已完成能力

- 公开文档包的受控读取与来源登记。
- 候选 Trial Fact Sheet、文档依赖关系和问题清单生成。
- 对模型输出执行独立 Gold Scorer，而不是由生成模型自行宣布正确。
- 输出 TP、FP、FN、TN、Precision、Recall、F1 和负对照准确率。

### 2.2 初始测试结果

| 指标 | 结果 |
|---|---:|
| TP | 4 |
| FP | 2 |
| FN | 4 |
| TN | 1 |
| Precision | 0.6667 |
| Recall | 0.5000 |
| F1 | 0.5714 |
| 负对照准确率 | 0.5000 |

模型正确发现了饮水要求冲突、研究人群/范围需人工复核、变更传播问题和治理门禁问题；但漏掉了给药间隔数值差异、住院时长差异、筛选窗口包含边界差异和完整追溯链问题。同时，它把合法的 Protocol/CRF 双时间轴映射误报为冲突。

因此，这条工作流属于**技术链路完整、识别性能中等、尚需改进规则和语义判别**，不能表述为已经完成高准确率审核。

### 2.3 2026-07-18 确定性规则突破

在不重新调用生成模型、复用同一冻结语义审核结果的条件下，新增了严格数值边界、跨来源时长与筛选窗口、完整来源链检查，并在两个时间轴事实均已批准时抑制已知的合法映射误报。独立 Gold Scorer 重评分结果为：

| 指标 | 初始值 | 新值 |
|---|---:|---:|
| Precision | 0.6667 | 0.8889 |
| Recall | 0.5000 | 1.0000 |
| F1 | 0.5714 | 0.9412 |
| 负对照准确率 | 0.5000 | 1.0000 |

10 项 gold tests 的期望判断现均正确；仍有一条未匹配的模型提醒，即对 `32 h → 36 h` 变更缺少药代学理由，因此保守计为一个 false positive。该结果证明规则层带来了真实提升，但案例规模仍小，不能外推为普适准确率。

## 3. Metformin-PAD 受控生成工作流

### 3.1 已完成能力

- Phase 1 只读取 `01_AI_VISIBLE_PHASE1`，生成开发问题、证据矩阵、候选事实、Synopsis 和章节草稿。
- Phase 2 只增加 `02_AI_VISIBLE_PHASE2`，执行事实调和、九章节增量修订、SoA/SAP 摘要/注册策略等关联产物生成。
- 隐藏参考资料仅在冻结生成结果后交给盲评器。
- 每次运行保存输入摘要、提示词摘要、模型、checkpoint、来源边界与审计记录。
- API 密钥支持从仓库外部 `.env` 文件加载，避免进入运行产物。

### 3.2 原始盲评为何不可信

第一次盲评中，语言模型给出 85 分，而按其分项 rubric 重新计算得到 98.18 分。该结果表面很高，但随后由三个相互独立的模拟审核员分别从医学安全、统计和注册质量角度复核，发现多项严重问题。这说明原始 LLM Judge 存在明显的宽松评分和自洽性偏差。

### 3.3 模拟专业审核员的主要发现

**医学与安全审核：**

- “所有随机化受试者”的 estimand 与要求至少用药一次且有基线后数据的 FAS 不一致。
- 死亡、截肢和血运重建不能简单作为 MAR 缺失值统一插补。
- AE 分级和部分安全管理规则存在错误或跨章节冲突。
- 研究停止规则、DMC 文件时间和造影剂停药规则仍需专业修订。

**统计审核：**

- 在双侧检验、效应差 20 m、标准差 65 m、85% power、15% 脱落的生成假设下，220 例无法复现；正态近似需要约 447 至 448 例，220 例对应 power 约 55.7%。
- 20 m 效应假设证据不足，MMRM 与 MI 的实现描述不完整。
- 多重性控制在不同章节存在直接矛盾。

**注册与质量审核：**

- 测试包中的模拟 FDA/CDE 沟通纪要被下游文本错误提升为监管认可或要求。
- 中国人群数量、桥接 PK、505(b)(2) 路径和 NMPA 2.4/CTA 依据不足。
- 部分来源定位和 TBD 管理尚不足以形成申报级追溯链。

以上审核均为模拟审核，不构成医学、统计、注册或质量专业批准。

### 3.4 新增确定性机器门禁

为避免语言模型用流畅文本掩盖基础矛盾，Phase 2 现增加不可被 LLM Judge 覆盖的确定性检查：

- `DET-STAT-001`：样本量和 power 的可复现性检查。
- `DET-REG-001`：合成监管材料被表述为真实监管认可的检查。
- `DET-STAT-002`：estimand 人群与分析集定义一致性检查。
- `DET-STAT-003`：多重性策略跨章节矛盾检查。
- `DET-STAT-004`：死亡等终末事件被不当纳入 MAR 插补的检查。

对冻结的 Phase 2 产物重新执行机器门禁后，运行状态由表面的高分结果改为：

```text
status: blocked_machine_gate
quality_gate: blocked_missing_evidence
```

本次冻结产物实际命中 `DET-STAT-001`、`DET-REG-001`、`DET-STAT-002` 和 `DET-STAT-004`。这意味着系统现在会诚实地阻止该方案进入专业人员批准阶段，而不是继续展示误导性的“98.18 分”。

## 5. 下一阶段优先事项

1. 为 NCT 工作流补强数值边界、时间轴映射和跨文件来源定位，降低 FN 与 FP。
2. 将样本量计算、estimand、多重性、安全事件和区域注册策略扩展成结构化校验器。
3. 为每条事实建立精确到文件、页码、章节、表格或单元格的证据定位。
4. 引入合格医学、统计和注册人员的人工复核结果，校准模拟审核员和机器门禁。
5. 保留所有模型版本、提示词摘要、输入摘要、人工修订和评分记录，支持可复现对比。

## 6. 对外表述边界

可以表述为：TrialCompiler 已搭建一个具有分阶段资料隔离、事实治理、文档依赖、增量修订、独立 benchmark 评分和机器门禁的临床文档工程原型，并在公开与合成案例上完成端到端复现。

不应表述为：系统已经能够自动设计临床试验、通过监管审核、替代专业人员，或已获得 98.18 分的临床质量证明。
