# TrialCompiler 方案说明

> 2026-07-17 收敛定位：根据医学资料包与竞品分析，本方案不再被表述为普通“AI 写临床方案”工具，而是定位为面向临床试验方案及其关联文件的 **结构化事实治理、依赖追踪与变更影响分析系统**。直接竞品已经证明协议结构化、来源追踪、跨文件一致性和修订协同是明确需求；TrialCompiler 的差异点应集中在中国药企场景下的确认事实层、规则/经验治理、任务闭环、可量化文档测试和与既有文档/运营系统的互补集成。

## 1. 方案整体概述

本方案拟构建 **TrialCompiler：可编译、可测试、可追踪、可增量更新的临床文档工程系统**，聚焦临床试验方案设计阶段的文档编制、一致性审核与变更影响分析。同一项主要终点、评估时间、样本量、入排标准、给药方案或访视安排会重复出现在不同章节、表格和关联文件中；一项设计事实改变后，人工逐处定位和修改容易造成旧值残留、版本冲突和同步遗漏。TrialCompiler 先由 AI 从授权资料中提取候选事实，再由医学、统计、注册等专业人员确认，形成具有来源、版本和审核状态的 Trial Fact Sheet；系统据此建立事实到章节、表格和关联文件的依赖图，辅助分章节编制、文档测试、影响传播和红线报告生成。当前范围不包括患者病历筛查和临床数据清洗。

## 2. 整体架构与核心功能模块

系统由六个核心模块组成。第一，**候选事实提取与人工确认模块**，提取终点、时间点、样本量、研究人群、给药和访视等信息，只有经授权人员确认后才形成有效 Trial Fact Sheet。第二，**文档依赖图模块**，记录事实与章节、表格及关联文件之间的直接引用和语义依赖。第三，**规则约束与证据模块**，将法规指南、企业模板和 SOP 与项目事实分层管理，并保留来源、版本、权限和适用范围。第四，**分章节辅助编制模块**，基于已确认事实生成候选文本，不自行补写未确认的研究设计。第五，**文档单元测试与变更影响模块**，结合确定性规则和语义检查发现跨章节冲突、旧值残留和缺失内容，并在事实变化时生成影响矩阵、候选红线和证据表。第六，**专家经验治理模块**，将人工确认的审核意见压缩为具有适用条件、反例、审批状态和有效期的经验卡片。飞书 Aily 负责资料上传、任务澄清、人工确认和审核协作，TrialCompiler 后端负责受控编译与测试。

## 3. 核心创新点：技术、模式与流程创新

技术上，本方案的核心不是“多 Agent 数量更多”，而是六项可检验的系统创新。第一，以 **Clinical Semantic Element** 为知识单元，在语义内容外增加权威性、版本、适用范围、审批状态和访问权限。第二，采用“高召回粗筛—元数据硬门控—语义等价精判”的治理感知检索，防止相似但过期、越权或错范围的经验进入上下文。第三，以事实依赖图实现**增量编译**，通过图可达性计算事实变化的影响范围，只重建受影响文档单元。第四，以 **Experience Compiler** 将审核轨迹转化为有条件、有反例、可撤销的程序性经验，而不是直接保存对话答案。第五，由生成/修复 Agent 与独立评价 Agent 形成有上限的 Active Compilation 循环，并以确定性测试、语义测试和人工质量门共同控制输出。第六，建设 TrialDocBench 与 Replay Harness，从事实提取、知识准入、缺陷识别、影响传播、修订接受率和经验迁移六个层级进行量化和消融实验。Cortex 的 Semantic Element、粗到细检索和缓存生命周期为系统提供方法启发；面向临床事实治理、依赖传播、人工审批经验与文档测试的组合扩展是 TrialCompiler 的主要贡献。

## 4. 竞品分析后的验证重点

竞品分析说明，单纯生成初稿、摘要和局部 QA 已经不足以构成强差异。后续原型与答辩应优先证明以下能力：

1. **确认事实层优于普通长文本生成**：同一个终点、时间点、样本量或入排标准只允许从 Trial Fact Sheet 的有效版本进入文档，而不是由模型在不同章节中自由改写。
2. **依赖图优于人工全文检索**：当 `Week 12` 改为 `Week 16` 时，系统能定位正文、访视表、统计原则、知情同意书或其他关联文件中所有直接和语义相关位置。
3. **文档单元测试优于人工交叉审阅**：用数字一致性、语义一致性、统计一致性和执行一致性四类测试输出可复现缺陷清单。
4. **经验治理优于保存聊天记录**：专家意见必须带有适用范围、来源、反例、审批状态和有效期，避免把某个项目的特殊批注意见误当成通用规则。
5. **系统互补而非替代 Veeva、太美、飞书等平台**：TrialCompiler 输出事实表、影响矩阵、红线稿、证据包和审计日志，可作为语义治理层接入现有受控文档和协作系统。
