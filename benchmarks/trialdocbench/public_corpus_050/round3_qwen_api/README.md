# Round 3 — Qwen API blind evaluation

> **已废弃（superseded）**：本目录的v1数据把人工变异值误设为权威事实，方向与“ClinicalTrials.gov冻结原值为正确基线”的项目约定相反。结果仅保留用于审计，不得用于报告结论。修正版见相邻目录 `round3_qwen_api_v2/`。

本目录保存第三轮丰富缺陷集的真实 Qwen API 评测产物。模型通过阿里云百炼 DashScope 的 OpenAI 兼容接口调用，模型名为 `qwen-plus`。运行时从仓库外部环境变量读取密钥，仓库内不保存任何凭据。

## 冻结评测设计

- 输入：同一临床试验、同一字段、同一受控文档单元中的“冻结权威事实”和“候选文档陈述”。
- 盲法：请求载荷不包含期望标签，也不说明样本是否经过变异。
- 校准：在校准集抽取 4 条，仅检查接口连通性、JSON格式和字段可解析性。
- 测试：校准后冻结系统提示词，独立运行测试集 160 条，不根据测试答案调参或重跑。
- 范围：评估受控字段值一致性识别和精确替换能力，不等同于外部专家的临床语义裁决。

## 冻结测试结果

- 请求：160；完成：160；错误：0；独立请求编号：160。
- 混淆矩阵：TP=79，FP=0，FN=1，TN=80。
- 准确率：99.375%（Wilson 95% CI：96.545%–99.890%）。
- 精确率：100%（Wilson 95% CI：95.363%–100%）。
- 召回率：98.750%（Wilson 95% CI：93.254%–99.779%）。
- F1：99.371%。
- 正确修订值：159/160。
- 墙钟时间：88.475秒；平均单请求延迟：3.164秒。
- token：提示词35,351，补全23,637。

唯一假阴性来自 `primary_outcome_measure` 字段：受控变异只在权威值尾部增加了 `[revised]`。模型将其解释为版本元数据而非结局指标正文，因而判定无需修订。严格字段相等标准下该判断计为假阴性；后续基准修订应以实质性结局指标变异替代此类弱元数据后缀，但本次冻结测试结果保持原样，不进行测试后调参。
