# Round 3 corrected — Qwen API evidence evaluation

本目录保存方向修正后的第三轮真实Qwen API评测。ClinicalTrials.gov冻结登记原值始终作为权威事实；缺陷候选由另一项真实试验的同类字段值移植得到，不再使用`[revised]`或`plus 1 day`等弱字符串变异。

## 设计

- 50个冻结ClinicalTrials.gov案例，8类字段。
- 400个跨试验同字段错配缺陷，400个未修改负对照。
- Qwen输入包含8字段登记证据片段、目标字段名和候选文档陈述；不包含金标签。
- 模型必须从带干扰字段的登记片段中找到目标字段，再判断候选值并给出修订值。
- 校准集16条覆盖8类字段；校准后冻结提示词和评分规则。
- 测试集160条，80个缺陷、80个负对照。

## 真实API测试结果

- 模型：`qwen-plus`；接口：DashScope OpenAI兼容`chat/completions`。
- 请求160，完成160，错误0，唯一请求编号160。
- TP=80，FP=0，FN=0，TN=80。
- 准确率、精确率、召回率、F1均为100%。
- 准确率Wilson 95%置信区间：97.655%–100%。
- 精确修订值：159/160（99.375%）。
- 墙钟时间181.807秒；平均单请求延迟4.498秒。
- 提示词58,151 token；补全25,254 token。

唯一修订值差异发生在主要结局时间范围。登记原文为`Baseline up to approximatly 24 months`，模型正确识别候选值有误，却将登记中的拼写错误`approximatly`自动改为`approximately`。严格逐字追踪标准下计为修订值错误；缺陷分类仍然正确。冻结测试结果保留，不做测试后调参。

## 结论边界

该结果证明模型能够从结构化登记证据片段中提取指定字段，并识别受控跨试验字段错配。它不等于开放式临床语义审核准确率，不代表对自然发生缺陷、完整研究方案、统计分析计划或跨文档依赖的专家级评测结果。
